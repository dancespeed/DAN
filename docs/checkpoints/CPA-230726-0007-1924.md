Данный файл предназначен для восстановления полного инженерного контекста проекта.

# Основные принципы проекта

Разработка ведётся исключительно архитектурно.

Обязательные правила:

1. Архитектура обсуждается до написания кода.
2. Работа ведётся кирпичик за кирпичиком.
3. Не перепрыгивать через этапы.
4. Любое изменение сначала объясняется.
5. Предоставляются полные файлы.
6. Пользователь самостоятельно заменяет файлы.
7. Посторонние рефакторинги запрещены.
8. Все решения должны соответствовать DPAS.

Конечная цель:

```
Producer
      ↓
MessageBus
      ↓
MessageDispatcher
      ↓
Receiver
```

Event после завершения миграции должен быть полностью удалён.

---

# Текущее переходное состояние

Пока система работает в режиме совместимости.

```
Legacy Producer
        ↓
     EventBus
      ├──────────────┐
      │              │
      ▼              ▼
 Event Queue     MessageBus
      │              │
      ▼              ▼
EventDispatcher  MessageDispatcher
```

Таким образом одновременно существуют два контура обработки.

---

# Причина выбранной схемы

Отказ от адаптеров.

Вместо создания отдельных мостов принято решение использовать EventBus как временный шлюз совместимости.

Именно EventBus выполняет преобразование:

```
Event
    ↓
Message
```

После завершения миграции EventBus также будет удалён.

На момент чекпоинта существуют четыре производителя событий.

```
System

Logic
PWM
```

Новые модули использовать Event не должны.

# Message

Контракт сообщения утверждён.

Структура сообщения фиксирована.

```
Header

Sender
Receiver
Attribute
MessageId

Payload

ObjectId
Value
```

Размер сообщения фиксирован.

Дальнейшие изменения формата не планируются без отдельного архитектурного обсуждения.

---

# MessageDispatcher

На данном этапе MessageDispatcher уже встроен в главный цикл.

Порядок обработки:

```
Input

↓

EventDispatcher

↓

MessageDispatcher

↓

PWM
```

Фактически новый транспорт уже работает.

Пока обработка сообщений ещё не используется прикладными модулями.

# Выполненный кирпичик

Выполнена интеграция MessageDispatcher в Runtime.

Проверено:

- проект успешно собирается;
- поведение платформы не изменилось.

Данный кирпичик считается завершённым.

---

# Следующий кирпичик

Начать миграцию первого потребителя.

Порядок миграции обязателен.

Для каждого модуля:

1.

Добавить обработку Message.

2.

Переключить модуль на Message.

3.

Проверить работу.

4.

Только после успешной проверки удалить обработку Event.

После этого переходить к следующему модулю.
